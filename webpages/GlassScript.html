Introduction to GlassScript

GlassScript is a high-level, human-readable automation language designed for GlassOS. It bridges the gap between natural language and system commands, allowing you to orchestrate complex workflows across multiple applications with zero boilerplate.

Contextual

Automatically detects which app is in focus to provide smart defaults.

Synchronous

Commands execute in perfect sequence, adhering to system timing.

Non-Destructive

Atomic operations ensure your data is always safe even in failed runs.

Basic Syntax

The Core Philosophy

GlassScript uses the tell app pattern. You define a target application, and all commands inside that block are interpreted by that app's specific runtime.

The "Tell" Block

tell app "Notepad"
    -- Your commands go here
end tell

Use double dashes for single-line comments.

Application Specifics

Notepad & GlassWord

Handle rich text manipulation, formatting, and layout control.

tell app "GlassWord"
    set font size to 16
    set style to bold
    align center
    write "GlassOS Status Report"
    insert newline
    set style to normal
    align left
    write "Generated via GlassScript"
end tell

Sheets

Target cells, ranges, and formulas for data analysis.

tell app "Sheets"
    select cell "A1"
    set value to "Revenue"
    select range "A2:A10"
    fill random 100 500
    set formula at "A11" to "=SUM(A2:A10)"
end tell

Calendar

Automate event creation and schedule management.

tell app "Calendar"
    create event with title "Neural Sync Meeting"
    set start time to "14:00"
    set end time to "15:00"
    invite "admin@glass-os.sys"
end tell

Mail

Compose, reply, and send messages through the mail protocol.

tell app "Mail"
    compose email to "client@neural.link"
    set subject to "System Update Log"
    set body to "The system migration is complete."
    send message
end tell

Database

Direct query access to System and User nodes.

tell app "Database"
    query "users" where "status=active"
    sync shard "messaging_primary"
    backup records locally
end tell

System Global Commands

These commands work outside of app blocks and control the execution flow.

notify "Message"

Triggers a system-wide toast notification.

wait 1000

Pauses execution for N milliseconds.

system.date

Injected variable for current system time.

run "app_id"

Force opens a background application.
